Investment Tracker — Architecture Approaches

How should we structure the data and page layout?

A

Single Page with Tabs

One /admin/investments page with 3 tabs:

  • Overview — Summary cards + combined growth chart for all FDs
  • Fixed Deposits — Table of all FDs with CRUD actions
  • Analytics — Daily growth, per-FD breakdown, projections

Pros

  • Everything in one place
  • Quick switching between views
  • Matches your existing analytics page pattern

Cons

  • Could feel crowded as FDs grow

⭐ Recommended — fits your current admin layout and keeps things focused

B

Two Separate Pages

Split into /admin/investments (management) and /admin/investments/analytics (visualization):

  • Investments page — FD table + summary cards + create/edit
  • Analytics page — All charts, daily growth, projections

Pros

  • Clean separation of management vs analytics
  • Room to grow each page independently

Cons

  • More navigation needed
  • Two sidebar entries
C

Integrated into Existing Analytics

Add an "Investments" section to the existing /admin/analytics page:

  • New section below the current fund projection chart
  • FD management via a modal or separate sub-route

Pros

  • No new pages needed
  • All analytics in one place

Cons

  • Analytics page gets very long
  • FD management feels shoehorned in
